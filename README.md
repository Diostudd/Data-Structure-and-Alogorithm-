Python Data Structures and Algorithms

This repository contains a collection of Jupyter Notebooks demonstrating fundamental Data Structures and Algorithms implemented in Python. It serves as an excellent reference for understanding how these core computer science concepts work under the hood.
📂 Repository Contents

The repository is organized into specific topics, each contained within its own Jupyter Notebook:
1. Linked Lists (Linked_list.ipynb)

Detailed implementations of node-based data structures.

  Singly Linked List: Operations including insertion (beginning, end, after node), deletion (beginning, end, by key), searching, and traversing.

   Doubly Linked List: Implementation with prev and next pointers, featuring forward and backward traversal alongside standard insertion and deletion methods.

2. Stacks and Queues (Stacks and Queues.ipynb)

Implementations of LIFO (Last-In-First-Out) and FIFO (First-In-First-Out) structures.

   Queues: Includes a basic Array Queue, a List-based Queue, and a fixed-size Circular Queue with O(1) operations. Also demonstrates Python's built-in collections.deque.

   Stacks: Custom Stack class with bounds checking.

   Stack Applications: Includes algorithms for Postfix expression evaluation, Valid Parenthesis checking, and Infix-to-Postfix conversion.

3. Trees (Trees.ipynb)

Comprehensive coverage of various tree data structures and traversals.

   Binary Trees: Basic tree structures and traversal algorithms.

   DFS (Depth-First Search): Preorder, Inorder, and Postorder traversals.

   BFS (Breadth-First Search): Level-order traversal using a queue.

  Expression Trees: Evaluating mathematical expressions stored in a tree structure.

   Binary Search Tree (BST): Implementations for insertion, searching, and complex deletions (handling nodes with zero, one, or two children).

   AVL Tree: A self-balancing binary search tree featuring height updates, balance factor calculations, and automatic Left/Right rotations to maintain O(logn) operations.

4. Searching and Hashing (Searching and Hashing.ipynb)

Algorithms for finding data and resolving hash collisions.

   Searching Algorithms: Linear Search (finding first or all occurrences) and Binary Search.

   Hash Tables: Custom implementations demonstrating various collision resolution strategies:

   Linear Probing

   Double Hashing

   Separate Chaining

5. Sorting (Sortings.ipynb)

Algorithms for ordering data.

    Bubble Sort: Implementation of the classic comparison-based sorting algorithm, demonstrating iterative swapping of adjacent elements.
